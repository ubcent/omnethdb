## Summary

Create a repeatable benchmark harness so memory-quality and curator-review signals can be judged on realistic evidence-bearing scenarios instead of intuition or generic similarity tasks.

## Goal

Make duplicate detection, update-target suggestion, synthesis-review clustering, and false-positive control measurable through a stable benchmark pack.

## Scope

In scope:

- curated fixtures for duplicate, near-duplicate, update, related, unrelated, and synthesis-review scenarios
- contradiction-like and noisy fixtures for downgrade or false-positive control
- repeatable harness or documented command
- mode-separated reporting for benchmark outputs

Out of scope:

- replacing the default embedder automatically
- benchmarking for generic semantic search outside OmnethDB memory scenarios
- production auto-tuning from benchmark results alone
