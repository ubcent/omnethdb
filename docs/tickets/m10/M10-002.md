## Summary

Extend inspector diagnostics so memory curators can review the supporting context behind duplicate and update candidates instead of relying on opaque signals alone.

## Goal

Help curators understand why a candidate surfaced by exposing focused supporting evidence, lineage context, and downgrade explanations in the inspector.

## Scope

In scope:

- supporting snippet or excerpt pane for duplicate diagnostics
- supporting snippet or excerpt pane for update diagnostics
- visible actor, time, latest-state, forgotten-state, and lineage context where relevant
- downgrade explanations for ambiguous, weak, or noisy support

Out of scope:

- editing memories directly from the inspector
- automatic cleanup actions
- changing retrieval ranking based on inspector evidence panes
- introducing hidden relation creation from inspector output
