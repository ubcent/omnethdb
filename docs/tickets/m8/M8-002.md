## Summary

Implement advisory similarity linting for memory writes so agents and operators can avoid duplicate roots and missed updates before the live corpus degrades.

## Goal

Make `remember` capable of returning quality signals based on similarity to the live corpus while preserving current write semantics.

## Scope

In scope:

- similarity lookup against live memories in the same space
- warning generation for duplicates and update candidates
- write-path integration that returns warnings without silently mutating behavior
- coverage for `static` memories first, with room for later expansion

Out of scope:

- automatic write rejection
- changes to recall semantics
- automatic merge, split, or forget actions

2. Obvious duplicate candidates yield `possible_duplicate`.
3. Changed-but-related candidates can yield `possible_update_target`.
