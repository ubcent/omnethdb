## Summary

Create a repeatable evaluation harness so embedding changes can be judged on real memory-quality tasks instead of intuition.

## Goal

Make embedding quality measurable for OmnethDB use cases such as duplicate detection, update-target suggestion, and false-positive control.

## Scope

In scope:

- curated eval fixtures for duplicate, update, related, and unrelated memory pairs
- harness or test command that computes quality metrics
- baseline results for the current embedder

Out of scope:

- shipping a new embedder by default
- benchmarking for generic semantic search outside OmnethDB memory scenarios
- auto-tuning thresholds in production at runtime
