## Summary

Give operators an explicit workflow to validate, compare, and apply runtime config changes to persisted spaces without destructive resets.

## Goal

Make governance and runtime-policy changes operationally safe for long-lived workspaces.

## Scope

In scope:

- config validation command
- config diff command against persisted space state
- config apply command that updates persisted space config explicitly
- verification for populated spaces

Out of scope:

- GUI config editing
- implicit config auto-sync on every startup
- schema migrations unrelated to runtime config
