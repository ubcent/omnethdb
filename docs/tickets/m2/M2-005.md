## Summary

Assemble lineage and governance semantics into the first real end-to-end write API.

## Goal

Implement `Remember` so root writes, updates, and promotion flows behave transactionally and leave no partial state behind.

## Scope

In scope:

- `Remember` contract for root and update writes
- integration of lineage rules and policy checks
- transactional write behavior across storage primitives
- failure behavior for invalid writes

Out of scope:

- derived writes
- forget and revive
- optimistic locking conflicts
